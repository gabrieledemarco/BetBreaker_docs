what it does

A statistical edge engine
for F1 betting markets

BetBreaker ingests real-time F1 session data, trains an ensemble ML model per season, and computes the probability gap between what the model believes and what the bookmaker prices — surfacing positive expected value opportunities.

📡
Real-time Session Data
Automatically pulls FP1/FP2/FP3 lap times, sector gaps, and qualifying deltas from OpenF1 and Jolpica/Ergast APIs — no API key required.
🤖
Ensemble ML Engine
Weighted ensemble of Logistic Regression, Random Forest and Gradient Boosting. Weights tuned via 5-fold cross-validation on ~6,000 synthetic F1 races calibrated on 2018–2025 data.
📊
Edge Detection
Converts bookmaker odds to implied probability, subtracts ensemble probability, and ranks all markets by statistical edge. Green >15%, Yellow 8–15%, Red below threshold.
🎯
Multi-builder
Automatically constructs high-edge accumulators (multiples) from the top-ranked selections, with combined probability and expected value calculations per combination.
📈
Ex-Post Evaluation
After race day, fetches official results via Jolpica API and evaluates each prediction — computing Accuracy, Brier Score, AUC-ROC, ROI by edge threshold, and calibration bias.
☁️
Cloud Persistent Storage
Predictions survive server restarts. Each season is stored in isolation to prevent cross-year model contamination — longitudinal accuracy tracking built-in.
how to use

Five steps from odds
to edge analysis

The workflow is designed to be completed in under five minutes per Grand Prix weekend.

01
Select the Grand Prix
In the 📡 Session Data tab, choose the year and round number. BetBreaker fetches live data from all three free practices and qualifying — no manual input required.
02
Enter bookmaker odds
In 📊 Inserisci Quote, paste odds as free text (e.g. “Russell vs Norris 1.08”) or upload a screenshot — Claude Vision will extract the markets automatically. A built-in demo is available for first-time users.
03
Run the ML analysis
Click Analizza con ML in the 🤖 Analisi ML tab. The engine builds 11 features per driver pair, runs the ensemble, and renders the edge chart with AUC and feature importance.
# Features used by the ensemble features = [ "grid_pos", "quali_gap", "avg_fp_gap", "best_fp_gap", "fp_consistency", "trend_improvement", "constructor_strength", "driver_experience", "session_issues", "fp2_gap", "sprint_gap" ]
04
Review report and multiples
The 📋 Report & Multiple tab shows a full ranked table sortable by edge, plus five auto-constructed accumulators. Download the full CSV for your records.
05
Evaluate predictions post-race
After the race, use 📈 Valutazione Ex-Post to pull official results, match predictions, and compute ROI and calibration metrics — building a longitudinal accuracy record over the season.

statistical foundations

Academic references

The models and evaluation methodology in BetBreaker are grounded in peer-reviewed literature. Below are the key references for each system component.

LR
Logistic Regression for Binary Classification
Cox, D.R. (1958) · Hosmer & Lemeshow (2000)
The base learner in the ensemble. Models the log-odds of driver D1 finishing ahead of D2 as a linear combination of 11 engineered features. Regularization C=0.5 (L2 penalty) controls overfitting on the sparse F1 feature space.
P(D1 > D2) = σ(β₀ + β₁·grid_gap + β₂·quali_gap + … + β₁₁·constructor_strength)
RF
Random Forests — Ensemble of Decision Trees
Breiman, L. (2001) · Machine Learning 45(1), 5–32
400-tree forest with bootstrap aggregation and random feature subsampling at each split. Highest ensemble weight (0.38) due to superior Top-6 AUC of 0.784. Provides built-in feature importance via mean decrease in impurity.
f̂(x) = (1/B) Σᵇ Tᵇ(x)   B=400, each Tᵇ on a bootstrap sample
GBM
Gradient Boosted Trees — Stagewise Additive Modeling
Friedman, J.H. (2001) · Annals of Statistics 29(5), 1189–1232
300 boosting stages with learning rate η=0.04 and max depth 3. Sequentially fits shallow trees to the negative gradient of binary cross-entropy. Ensemble weight 0.37. Particularly sensitive to the trend_improvement feature.
F_m(x) = F_{m-1}(x) + η · h_m(x)   h_m = argmin Σ L(y_i, F_{m-1}(x_i)+h(x_i))
ENS
Weighted Probability Ensembling
Dietterich, T.G. (2000) · Multiple Classifier Systems, LNCS 1857
Final probability is a weighted linear combination of three base learners. Weights (LR=0.25, RF=0.38, GBM=0.37) selected by grid search on 5-fold cross-validation to maximize AUC on held-out race data.
P_ensemble = 0.25·P_LR + 0.38·P_RF + 0.37·P_GBM
CAL
Probability Calibration — Isotonic Regression
Zadrozny & Elkan (2002) · KDD ’02 · Niculescu-Mizil & Caruana (2005) · ICML
Post-hoc calibration applied once ≥20 evaluated real predictions are available. Isotonic regression maps raw ensemble scores to empirically calibrated probabilities, reducing overconfidence bias measured via reliability diagrams.
P_cal = iso(P_ensemble)   s.t. iso non-decreasing, minimizes Σ(P_cal_i − y_i)²
EV
Kelly Criterion & Expected Value in Prediction Markets
Kelly, J.L. (1956) · Bell System Technical Journal 35(4) · Brier, G.W. (1950)
Edge = P_model − P_implied where P_implied = 1/odds. Brier Score measures mean squared error between predicted probabilities and binary outcomes (target <0.20). ROI simulation uses flat-stake betting across all green-edge predictions.
Edge = P_ensemble − (1/odds)  ·  BS = (1/n) Σ_i (P_i − y_i)²
